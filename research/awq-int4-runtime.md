## 1. Introduction

Activation-aware Weight Quantization (AWQ, Lin et al. 2023) is the de-facto
method for turning a 7–8B-class FP16 LLM into a deployable INT4 model: it
protects the weight channels aligned with large-magnitude activations by
scaling them up before quantization, then compensating the activations. The
production toolchain, however, is largely black-box — AutoAWQ / GPTQ packages
that hide the scale search, the packing, and the runtime handoff behind a
single call.

We wanted the opposite: a **from-scratch, pure-PyTorch, model-agnostic**
implementation where every stage — activation statistics, per-layer scale
search, INT4 packing, reconstruction verification, and the bridge to a real
INT4 runtime — is inspectable. And we wanted to prove, end-to-end on a
viable-sized model on CUDA, that it produces a **4× smaller model with
near-zero loss**.

**The accomplishment.** On Qwen3-8B (the 7B-class target; Qwen3-7B is not
published on the Hub), our pipeline produces an INT4 model that is **4.0×
smaller** in linear weights and runs at **1.034× FP16 perplexity** in a real
INT4 GEMM runtime, with coherent greedy generation indistinguishable in
substance from the FP16 baseline.

**Contributions.**
1. A pure-PyTorch, model-agnostic AWQ pipeline (`calibrate / scales / quantize
   / verify`) that derives the transformer layout from the model rather than
   hardcoding layer counts, and streams `safetensors` one tensor at a time so
   the quantizer itself stays memory-safe.
2. `awq export` — a re-packer from our internal `quantized_state.pt` into the
   AutoAWQ / HF-AWQ GEMM on-disk format, with the **shared-per-norm-group**
   scale reconciliation and a `zero=7` signed→unsigned INT4 mapping that
   preserves our quantization grid with **no re-quantization error**.
3. A measured comparison of two AWQ scale-application strategies —
   **norm-folding** vs **weight-dequantization** — showing norm-folding is
   **20× more accurate per weight** and is the single decisive factor behind
   the near-zero-loss 8B result.
4. An end-to-end CUDA evaluation (perplexity + greedy generation) on Qwen3-0.6B
   / 1.7B / 8B that establishes the quality bar and the scaling trend.

## 2. Background

**AWQ scaling.** For a linear layer with weight `W ∈ ℝ^{d_out×d_in}` and
per-input-channel activation magnitude `x_max = |X|.mean(0)`, AWQ chooses a
per-channel scale

```
s = (x_max^α) / (mean(x_max)^α)            # α ∈ [0, 1), mean-normalized
```

scales the weights **up** by `s` before quantization (`W' = W·s`), and divides
the **activations** by `s` at inference. The reference implementation
(`mit-han-lab/llm-awq`) folds `1/s` into the **preceding norm**'s weight
(`norm.weight /= s`) rather than storing `s` per linear. The optimal `α` is
found by grid search, scored by the activation-weighted reconstruction error
`‖(W − Q(W·s)/s)·diag(x_max)‖²`; `α=0` gives `s=1` (plain RTN), so the search
can only improve on RTN, never worsen it.

**Group-wise symmetric INT4.** Weights are quantized in groups of `group_size`
input channels with one FP16 scale per group: `qscale = max|group|/7`,
`q = round(w/qscale).clamp(-7,7)`, so the signed grid is `[-7, 7]` (15 codes).
Two INT4 are packed per byte in the internal artifact.

**AutoAWQ / HF-AWQ GEMM on-disk format.** The runtime-loadable layout is
unsigned `[0,15]` int4 packed 8-per-int32:

```
qweight : int32 [d_in, d_out // 8]            # 8 int4 packed along the OUTPUT axis
qzeros  : int32 [d_in // group_size, d_out // 8]
scales  : fp16 [d_in // group_size, d_out]
dequant : (reverse_awq_order(unpack(qweight)) − zeros) * scales
```

with `AWQ_PACK_ORDER = [0,2,4,6,1,3,5,7]` governing the within-int32 ordering.

## 3. Method

```txt
                calibrate                scales                  quantize
FP16 model  ────────────────►  calibration_stats.pt  ────────────────►  awq_scales.pt
 + prompts    (running-sum       (|X|.mean(0)           (per-channel s
               hooks, O(d_in)       per linear)            [d_in] per layer,
               per layer)                                  α grid search)
                                                                   │
                                                                   ▼
prompts ───────────────────────────────────────────────────────────►│
                                                                    quantize
                                                  ┌─────────────────► (W·s, INT4 pack,
                                                  │                  2 int4/byte, CPU
                                                  │                  streaming)
                                                  ▼
                                         awq_quantized/ ──► export ──► awq_hf/ (AutoAWQ GEMM)
                                         quantized_state.pt             qweight/qzeros/scales
                                         metadata.json                  + folded-norm FP16
                                                  │                            │
                                                  ▼                            ▼
                                           verify (MSE)                 INT4 GEMM runtime
                                                                        (AutoAWQ) → PPL / gen
```

### 3.1 Calibrate
A forward hook on every `nn.Linear` aggregates the per-input-channel
activation magnitude **on-the-fly** as a running sum: `ci = x.abs().mean(0)`,
accumulated per layer with a running count. Memory is `O(d_in)` per layer —
**not** `O(samples × tokens × d_in)` — so hundreds of calibration samples cost
nothing beyond a single forward. We calibrate on 128 WikiText-2 samples at
`max_length = 2048`.

### 3.2 Scales
For each linear, we grid-search `α` over `n_grid = 20` values in `[0, 1)` and
score each candidate `s` by **actually quantizing** the weight with it
(`_pseudo_quantize_int4` mirrors the real quantizer's grid exactly) and
computing `‖(W − Q(W·s)/s)·diag(x_max)‖²`. `α=0` is RTN (`s=1`), so the search
never does worse than RTN. The layer count for depth-dependent skip strategies
is **derived from the calibration stats** (`model.layers.{i}.*`), not
hardcoded — the implementation is model-agnostic for any Llama/Qwen/Mistral-style
causal LM.

### 3.3 Quantize
The quantizer **never loads the full model**. It streams `safetensors` one
weight tensor at a time, applies `W·s`, performs group-wise symmetric INT4
quantization, packs two INT4 per byte, and writes `quantized_state.pt` +
`metadata.json`. Peak memory is one tensor plus its packed output. All 252
linear layers of Qwen3-8B are quantized this way.

### 3.4 Verify
`dequantize_layer` inverts the packer (`Q(W·s)/s`) and reports MSE against the
original FP16 weight read from disk. This is a reconstruction-quality check on
the quantization itself, independent of any runtime.

### 3.5 Export — the design behind near-zero loss
The hard part is producing a model a real INT4 runtime can load, because our
per-linear `s` does not map 1:1 onto the standard AWQ runtime convention:

- `q_proj / k_proj / v_proj` share `input_layernorm`;
- `gate_proj / up_proj` share `post_attention_layernorm`;
- you cannot fold three different `s` into one norm;
- `o_proj` and `down_proj` have **no preceding norm** at all.

The exporter reconciles this as follows:

1. **Aggregate** the per-linear `s` into one **shared scale per norm group**
   (geometric mean over the member linears) and **fold `1/s` into that norm**
   (`norm.weight /= s_shared`).
2. **Re-quantize** the 5 norm-following linears (`q/k/v/gate/up`) with the
   shared `s`.
3. **Export `o_proj` / `down_proj` as plain RTN INT4** (`s = 1`) — they have no
   preceding norm to absorb a scale.
4. Map the signed `[-7, 7]` grid onto AutoAWQ's unsigned `[0, 15]` grid by
   storing **`zero = 7`** and `q_unsigned = q_signed + 7`, so the runtime's
   `(q − zero) · scale` **exactly reproduces** our group dequant
   `q_signed · group_scale` — **zero re-quantization error** on the AWQ-scaled
   linears.

The output directory contains AutoAWQ GEMM `qweight / qzeros / scales` per
quantized linear, the FP16 (folded) norms + embeddings + `lm_head`, and a
`quantize_config.json` (`{bits:4, group_size, zero_point:true,
quant_method:awq, version:gemm}`). AutoAWQ loads it directly and replaces each
quantized linear with a `WQLinear` INT4 GEMM module.

## 6. Why near-zero loss holds: norm-fold vs weight-dequant

The single most important design choice is **how the AWQ scale `s` is applied
at inference**. There are two strategies:

- **Weight-dequantization:** dequantize each linear to `Q(W·s)/s` as an FP16
  weight and run a normal FP16 forward (no norm change).
- **Norm-folding:** store `Q(W·s)` and fold `1/s` into the preceding norm
  (`norm.weight /= s`), so the **activation** is divided by `s`. This is the
  reference `mit-han-lab/llm-awq` convention and what `awq export` does.

They are *not* equivalent. On `model.layers.0.self_attn.q_proj` (1.7B,
`group_size=128`, original weight norm = 72.03):

| Strategy | Weight norm | MSE vs FP16 | Per-layer output rel. error |
|---|---|---|---|
| Weight-dequant (`Q(W·s)/s`) | 79.38 (**+10%**) | 4.2e-4 | **29–46%** |
| **Norm-fold** (`Q(W·s)`, `s` in norm) | 72.27 (+0.3%) | **2.1e-5** | — |

**Mechanism.** Weight-dequantization divides by `s`. On channels where the
optimal AWQ scale `s < 1` (low-activation channels), the `1/s` factor
**amplifies** the INT4 rounding error — the group step is set by the group
maximum, but the small-`s` channels are quantized coarsely and then expanded
by `1/s`. This inflates the weight norm (+10%) and concentrates 29–46%
per-layer output error, which compounds across 28–36 transformer layers into
near-random output (end-to-end PPL ≈ 10⁵; see Appendix B).

Norm-folding applies `1/s` to the **activation** instead — the weight stays
`Q(W·s)`, exactly the value the quantizer optimized — so there is no `1/s`
amplification. The result is **20× lower per-weight MSE** and, end-to-end, the
**1.034× FP16** 8B number. This is the difference between "INT4 destroys an 8B
model" and "+3.4%."

## 7. Ablation — grid search vs fixed α

| Scale selection (1.7B) | Verify MSE (weight) | AWQ-runtime PPL |
|---|---|---|
| **Grid search** (`--n-grid 20`) | **2.74e-4** | 20.90 |
| Fixed α = 0.5 (`--no-grid-search`) | 5.63e-4 | **19.94** |

Grid search **wins on per-layer weight reconstruction** — ~2× lower MSE —
confirming the search is real and the scoring function is correctly weighted at
the per-layer level. Fixed-α **wins on end-to-end runtime PPL**, however, and
this is **not** a scoring bug: the export re-quantizes with a
*shared-per-norm-group* `s` (geometric mean of the member linears), so the
per-layer grid search's aggressive, layer-specific `s` does not fully propagate
through the shared-`s` aggregation. Fixed-α produces smoother per-linear scales
that aggregate into a better shared `s`.

**Future direction.** Grid search over **norm-groups** (a shared `s` per
`input_layernorm` / `post_attention_layernorm`) rather than per-linear — i.e.,
optimize the quantity the exportable format actually uses. This is the expected
path to push 1.7B toward the 8B near-lossless regime.

## 8. Discussion & limitations

- **The win, restated.** A from-scratch AWQ implementation produces a **4×
  smaller, +3.4%-PPL** Qwen3-8B that runs in a real INT4 GEMM runtime. This is
  production-relevant, not a toy result on a sub-viable model.
- **RTN for `o_proj` / `down_proj`.** These two linears per block have no
  preceding norm, so the exporter leaves them at RTN (`s=1`) rather than
  applying a scale it cannot fold. This is a documented conservatism; a
  follow-up can absorb their `s` into the preceding projection (`v_proj` /
  `up_proj` output channels) for further quality headroom.
- **Norm-group-level search.** The ablation shows per-layer grid search
  optimizes the wrong unit for the exportable format; norm-group search is the
  natural next improvement.
- **Generalization.** Evaluated on Qwen3; the `model.layers.{i}.*` convention
  makes Llama/Mistral support straightforward. `lm_head` is intentionally
  skipped (vocabulary projection is too sensitive to INT4).

---

## 9. Conclusion

A from-scratch, pure-PyTorch, model-agnostic AWQ implementation, with a
runtime-loadable export, delivers a **4× smaller Qwen3-8B at 1.034× FP16
perplexity** — near-zero loss, coherent generation, runnable in a real INT4
GEMM runtime. The decisive factor is **norm-folding** the AWQ scale
(20× more accurate per weight than weight-dequantization), and the clear next
step is **norm-group-level scale search** to bring smaller models into the
same near-lossless regime.

## Appendix A — AutoAWQ GEMM packing spec (for implementers)

```txt
qweight : int32 [d_in, d_out // 8]            # 8 int4 per int32, packed along d_out
qzeros  : int32 [d_in // group_size, d_out // 8]
scales  : fp16 [d_in // group_size, d_out]

packing : qweight = pack(apply_order(W_int4, AWQ_PACK_ORDER=[0,2,4,6,1,3,5,7]), "column")
dequant : iweight = reverse_awq_order(unpack_column(qweight))
          w = (iweight − zeros) * scales
```

Our signed `[-7,7]` grid maps to the unsigned `[0,15]` grid via `zero = 7`,
`q_unsigned = q_signed + 7`, so `(q − zero) · scale = q_signed · group_scale`
— the runtime dequant reproduces our group dequant exactly, with no
re-quantization error on AWQ-scaled linears. `o_proj` / `down_proj` are
exported as RTN (`zero = 7`, `s = 1`).
